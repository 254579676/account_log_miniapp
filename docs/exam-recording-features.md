# 考试记录功能增强

## 功能概述

为考试记录页面增加了两个新的录入入口：【单元测试】和【综合测试】，提供更精准的考试记录方式。

## 新增功能

### 1. 单元测试
- **入口**：红色渐变按钮，图标 📝
- **功能**：记录单科成绩
- **字段**：
  - 科目选择（必填）
  - 考试名称（自动生成或手动输入）
  - 得分/总分（必填）
  - 考试日期
  - 试卷照片
  - 备注

### 2. 综合测试
- **入口**：绿色渐变按钮，图标 📋
- **功能**：记录总分成绩，并可关联多个单科成绩
- **字段**：
  - 考试名称（必填）
  - 总分（必填）
  - 考试日期
  - 班级排名/年级排名（选填）
  - 单科成绩列表（可动态添加）
  - 试卷照片
  - 备注

### 3. 单科成绩管理
- **动态添加**：点击"添加科目"按钮
- **智能过滤**：自动过滤已添加的科目
- **独立录入**：每个科目独立的得分和总分
- **删除功能**：支持删除已添加的单科成绩

## 技术实现

### 1. 界面设计
- **三按钮布局**：统计、单元测试、综合测试
- **渐变配色**：
  - 统计：紫色渐变 (#667eea → #764ba2)
  - 单元测试：红色渐变 (#ff6b6b → #ee5a24)
  - 综合测试：绿色渐变 (#4ecdc4 → #44a08d)
- **响应式设计**：小屏幕下按钮垂直排列

### 2. 数据结构
```javascript
// 单元测试数据
unitTestData: {
  subject_id: '',
  subject_name: '',
  exam_name: '',
  exam_type: '单元测试',
  score: '',
  total_score: '100',
  exam_date: '',
  images: [],
  remarks: ''
}

// 综合测试数据
comprehensiveData: {
  exam_name: '',
  exam_type: '综合测试',
  total_score: '',
  exam_date: '',
  class_ranking: '',
  grade_ranking: '',
  images: [],
  remarks: '',
  subjects: [] // 单科成绩列表
}
```

### 3. 核心方法
- `onAddUnitTest()` - 打开单元测试弹窗
- `onAddComprehensiveTest()` - 打开综合测试弹窗
- `onAddSubjectScore()` - 添加单科成绩
- `onSubjectScoreChange()` - 修改单科成绩
- `onDeleteSubjectScore()` - 删除单科成绩
- `onSaveUnitTest()` - 保存单元测试
- `onSaveComprehensiveTest()` - 保存综合测试

### 4. 图片管理优化
- **多弹窗适配**：根据弹窗类型管理对应的图片数组
- **统一接口**：onChooseImage、onImagePreview、onDeleteImage
- **智能切换**：自动识别当前弹窗类型

## 业务流程

### 单元测试流程
1. 点击【单元测试】按钮
2. 选择科目（自动生成考试名称）
3. 输入得分和总分
4. 选择考试日期（默认今天）
5. 上传试卷照片（可选）
6. 填写备注（可选）
7. 保存记录

### 综合测试流程
1. 点击【综合测试】按钮
2. 输入考试名称和总分
3. 动态添加单科成绩
4. 为每个科目设置得分
5. 上传试卷照片（可选）
6. 填写排名和备注（可选）
7. 保存记录（自动创建主记录和单科记录）

## 数据存储

### 综合测试存储逻辑
1. **主记录**：保存综合测试的总分和基本信息
2. **单科记录**：为每个科目创建独立的考试记录
3. **关联关系**：通过parent_exam_id字段关联到主记录
4. **命名规则**：单科记录名称为"综合测试名称 - 科目名称"
